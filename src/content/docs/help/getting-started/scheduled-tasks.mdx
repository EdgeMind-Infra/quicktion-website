### 什么是定时任务？

就是给 Agent 设置一个执行计划：

- 每天早上 8 点，帮我跑一次经营分分析，有异常推送给我
- 每周一，帮我监控一次同城竞品的新视频和促销动态
- 每月 1 号，帮我检查货架上的商品状态，有下架的提醒我
- 每逢节假日前 3 天，帮我出一套节日营销方案供我参考

设置好了之后，Agent 按时自动执行，完成了通知你。你不需要记着，不需要手动启动。

### 能设置什么样的周期？

周期可以任意设置：

- **每天**：适合日常监控类任务，比如每天看一眼经营数据
- **每周**：适合竞品分析、内容选题等周频任务
- **每月**：适合阶段性总结、货架盘点等月度任务
- **自定义时间点**：比如每逢周五晚上跑一次，或者指定某个日期执行

你说什么周期，Agent 就按什么周期跑。

### 你睡着的时候，它在干什么？

举个例子。

你设置了：每天凌晨 2 点，帮我监控竞品最新动态，有重要信息早上 8 点推送给我。

凌晨 2 点，你早就睡了。Agent 自动启动，打开各个平台，扫描同城竞品的新发视频、新上活动、价格变化……全部整理成一份简报。

早上 8 点，你刚起床，微信或飞书就收到一条消息：昨晚竞品监控完成，发现 2 家同类门店上了新的节日套餐，详情请查看附件报告。

你喝着早茶，翻看报告，决定今天要不要跟进——所有信息收集和整理的工作，AI 已经帮你做好了。

### 哪些任务适合定时？

不是所有任务都适合定时，但以下这些特别适合：

- **持续监控类**：竞品动态监控、货架商品状态检查、经营分变化追踪
- **定期分析类**：周度/月度经营数据分析、达人合作效果复盘
- **节点提醒类**：节假日营销准备提醒、活动结束前的核销提醒
- **持续运营类**：每周内容选题建议、定期发布前的脚本生成

这些任务如果手动做，既容易忘、又费时间。定时设置好，一劳永逸。

### 任务完成了，怎么通知我？

结合 MCP 连接功能，任务完成的通知和成果，直接发到你的微信或飞书。

你不需要登录平台去查，系统主动找你。报告文档直接附在通知里，点开就看。

如果任务跑完了发现有问题——比如货架上有商品下架了——通知里会直接说明，你回复一句：帮我重新上架，Agent 立刻去处理。

### 设置复杂吗？

一点都不复杂。

在平台里对 Agent 说：帮我设置一个每周一早上 9 点运行的竞品监控任务，Agent 就帮你创建好了。你可以随时查看、修改、暂停、删除定时任务，操作就像在日历里添加一个提醒一样直观。

一次设置，长期受益，不用每次都想着：我应该去看看竞品了。
